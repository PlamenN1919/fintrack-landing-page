# 🌐 DNS Конфигурация за fintrackwallet.com в jump.bg

## 📋 Бърз преглед

Трябва да добавиш 2 DNS записа в jump.bg за да свържеш домейна с Vercel.

---

## 🎯 Какво трябва да направиш

### Стъпка 1: Влез в jump.bg
```
URL: https://www.jump.bg
→ Вход (горе вдясно)
→ Въведи username и парола
```

### Стъпка 2: Отвори DNS управление
```
Моите домейни → fintrackwallet.com → DNS управление
```

**Забележка:** В jump.bg може да се казва:
- "DNS управление"
- "DNS Zone"
- "Name Servers"
- "Управление на DNS зони"

---

## 📝 DNS Записи които трябва да добавиш

### Запис 1: A Record (за fintrackwallet.com)

```
┌─────────────────────────────────────────┐
│ Тип (Type):        A                    │
│ Име (Name/Host):   @                    │
│ Стойност (Value):  76.76.21.21          │
│ TTL:               3600 (или Auto)      │
└─────────────────────────────────────────┘
```

**Какво означава:**
- `@` = root domain (fintrackwallet.com)
- `76.76.21.21` = Vercel IP адрес

### Запис 2: CNAME Record (за www.fintrackwallet.com)

```
┌─────────────────────────────────────────┐
│ Тип (Type):        CNAME                │
│ Име (Name/Host):   www                  │
│ Стойност (Value):  cname.vercel-dns.com │
│ TTL:               3600 (или Auto)      │
└─────────────────────────────────────────┘
```

**Какво означава:**
- `www` = www субдомейн
- `cname.vercel-dns.com` = Vercel CNAME target

---

## ⚠️ ВАЖНО: Изтрий стари записи

**ПРЕДИ** да добавиш новите записи, изтрий всички стари A и CNAME записи за:
- `@` (root)
- `www`

Ако има стари записи, DNS няма да работи правилно!

---

## 🔍 Как изглежда в jump.bg интерфейса

След като добавиш записите, трябва да видиш нещо подобно:

```
┌──────────┬──────────┬──────────────────────┬─────────┐
│ Тип      │ Име      │ Стойност             │ TTL     │
├──────────┼──────────┼──────────────────────┼─────────┤
│ A        │ @        │ 76.76.21.21          │ 3600    │
│ CNAME    │ www      │ cname.vercel-dns.com │ 3600    │
└──────────┴──────────┴──────────────────────┴─────────┘
```

## ✅ Проверка

### Метод 1: Vercel Dashboard
1. Отвори Vercel → Settings → Domains
2. Refresh страницата на всеки 5 минути
3. Когато DNS propagate-не, ще видиш ✅ зелен чек

### Метод 2: DNS Checker
1. Отвори: https://dnschecker.org
2. Въведи: `fintrackwallet.com`
3. Избери "A" record
4. Трябва да видиш `76.76.21.21` в повечето локации

### Метод 3: Terminal (за напреднали)
```bash
# Провери A record
nslookup fintrackwallet.com

# Провери CNAME record
nslookup www.fintrackwallet.com
```

## 🐛 Troubleshooting

### Проблем: Не мога да намеря "DNS управление" в jump.bg
**Решение:** Потърси за:
- "DNS"
- "Name Servers"
- "Zone File"
- "Advanced DNS"
- Или контактирай jump.bg support

### Проблем: jump.bg не приема `@` за Name
**Решение:** Опитай:
- Остави полето празно
- Напиши `fintrackwallet.com` (пълния домейн)
- Напиши `.` (точка)

### Проблем: CNAME не работи
**Решение:** 
- Увери се, че си изтрил стари A записи за `www`
- CNAME и A record не могат да съществуват едновременно за същото име

### Проблем: DNS не propagate-ва след 1 час
**Решение:**
1. Провери дали записите са правилно въведени в jump.bg
2. Провери дали няма стари записи
3. Опитай да изтриеш и добавиш отново записите
4. Изчакай още малко (понякога отнема до 2-3 часа)

## 🎓 Допълнителна информация

### Какво е A Record?
A Record (Address Record) свързва домейн име с IP адрес.
```
fintrackwallet.com → 76.76.21.21 (Vercel сървър)
```

### Какво е CNAME Record?
CNAME Record (Canonical Name) свързва домейн с друг домейн.
```
www.fintrackwallet.com → cname.vercel-dns.com → 76.76.21.21
```

### Защо използваме CNAME за www?
Vercel може да промени IP адреса си. С CNAME, промяната е автоматична.
С A record, трябва ръчно да обновяваш IP-то.

### Какво е TTL?
TTL (Time To Live) определя колко време DNS записът се кешира.
- 3600 секунди = 1 час
- По-нисък TTL = по-бързи промени, но повече DNS заявки
- По-висок TTL = по-бавни промени, но по-малко DNS заявки
